git 的简单使用

发表于 2015-09-08   |   分类于 未分类

1.git init 将一个文件夹变成一个仓库 2.git add xxx 将告诉git,把文件添加到仓库(就像填表) 3.git commit -m "xxxxx" ,告诉git,把文件提交到仓库. 4.git status 查看当前状态 5.git diff 查看提交记录<!--more--> 6.git log 查看提交记录 7.添加ssh ssh-keygen -t rsa -C "youremail@example.com" 8\. 使用此命令将本地库与与远程库联系在一起git remote add origin git@github.com:michaelliao/learngit.git 9.git push -u origin master 第一次提交加上-u,git master分支推送到远程的master,以后的推送或者拉取就可以简 化命令 10.版本控制就在本地完成了所有的操作后将本地的代码推到远地 11.master 是指当前的主分支,head是指向主分支的,如果master变了,head会变 12.git checkout -b new branch 创建新的分支,它相当于git branch xxx git checkout xxx 13.git checkout -d xxx 删除分支 14.git stash 将状态保存起来 git stash apply 将状态恢复, git stash drop 将stash删除 git stash pop将状态恢复并把stash内容删除 git stash list 查看stash的状态 15.git merge xxx创建一个分支后将分支进行合并, 16.git config --global color.ui true 添加颜色

linux内存管理机制

发表于 2015-08-29   |   分类于 linux的学习记录

在Linux中经常发现空闲内存很少，似乎所有的内存都被系统占用了，表面感觉是内存不够用了，其实不然。这是Linux内存管理的一个优秀特性，在这方面，区别于Windows的内存管理。主要特点是，无论物理内存有多大，Linux 都将其充份利用，将一些程序调用过的硬盘数据读入内存，利用内存读写的高速特性来提高Linux系统的数据访问性能。而Windows是只在需要内存时，才为应用程序分配内存，并不能充分利用大容量的内存空间。换句话说，每增加一些物理内存，Linux都将能充分利用起来，发挥了硬件投资带来的好处，而Windows只将其做为摆设，即使增加8GB甚至更大。

Linux的这一特性，主要是利用空闲的物理内存，划分出一部份空间，做为cache、buffers ，以此提高数据访问性能。
