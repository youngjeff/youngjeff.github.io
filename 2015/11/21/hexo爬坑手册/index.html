hexo爬坑手册

发表于 2015-11-21   |   分类于 linux的学习记录

前言

我以前是用新浪的sae，加上wordpress，来构建自己的博客，
确实很简单，也很不错，但是对于一个程序员，我不满足简单的方式玩blog，
于是选择了hexo（基于nodejs）+markdown+github+atom(一款不错的编辑器)
来写自己的博客。

hexo的优点

  • markdown语法书写简单，
  • 基于hexo可以写离线的博客，再上传到github，在哪里都可以写自己的博客
  • github是免费的，在全球都有不错的访问速度

hexo类博客的不足

步骤：

  • hexo的安装具有难度，不是一个新手都可以玩的技术，可能初学者上手有难度，
  • hexo也不是随便哪台电脑都可以写的，每部署一台电脑，有点麻烦，nodejs的安装需要时间，hexo 需要时间，所以转hexo的新手最好有心理准备。

hexo，你好

先安装nvm：$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh
或者$ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
再安装nodejs： nvm install 0.10
就可以安装hexo：npm install -g hexo-cli

我们来建在自己的第一个离线博客吧

进入自己创建的hexo文件夹，初始化：
hexo init
配置：npm install
新建一篇文件：hexo new “title”
按照markdown的语言写好之后，可以用
生成文章： hexo g
然后预览： hexo s
确认自己写的很好，就提交： hexo d
